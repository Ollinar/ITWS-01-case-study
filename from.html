Form Tag

The easiest way to understand the form tag is to think of it as a container for all of the user input that the programmer wants to submit somewhere.

List of tags and attributes used inside form tag

Tag/Attributes Explanation
action Attribute used on the form tag to tell what webpage the form should be subbmited to.
method Attribute used on the form tag to tell how the form should be submitted. there are 2 main types of method, GET and POST. GET will put all of the things to be submitted on the URL thus its limited by the URL maximum limit. Meanwhile POST method wont put the things to be submitted on the url thus its not limited and more secure for things like passwords. By defult the form will use GET.
target Attribute used on the form tag to specify where should the response forom the form submition be displayed. "_blank" will open in new tab while "_self" will open it to the current tab
Input Input tag is what really makes up the Html form. It is the part where users actually interact and input their informations.
Label Label tag is used to label and input field so the user knows what that input is for.
Fieldset Fieldse tag is used to group inputs together. For example grouping inputs about personal information together on one fieldset.
Legend Legend tag is like label tag but for Fieldset. It is used to inform the user what a fieldset is for
for An attribute used on a label to link it to an inuput field by using the id of that input field. If tha label is linked to an input field, when the user clicks the label it'll put the user in the linked input field.
name An attribute used on input tag to give it a name or "key name". When a for is submitted, it'll be in a format of key value pairs. The key will be what the programmer used as a name, while the value is what the user inputted.
